14: Loading View

The next thing to do was to create a loading view that would show until all the assets had loaded.

Due to the loading view needing to load before the assets, I added it inline at the top of the page instead of loading it in separately so that it could display almost instantly while other assets were loading.

I decided to create a dark background and animate the star images (the same used as the cursor while the shake animation happens), however animate it slightly differently to signify a loading view.

Creating the loading images

In Figma, I got the original star image and separated each star into its own image so that each could be animated.

I then grouped them into a frame and exported each with the frame so that they would position correctly on the page.

I used the same positioning previously for other images to centre them in the page, however this was done inline so that on each star that there was no delay for the CSS to be applied.

                                
    style="top: 50%; left: 50%; transform: translate(-50%, -50%); position: absolute;"

Animating the loading view

Next, I created a fade in and out animation and set each to have this animation, but for each to have a different initial delay so that it created a unified and repeating loading icon while the page loaded.

                                                                    
    @keyframes load {
        0% {opacity: 100%;}
        25% {opacity: 30%;}
        100% {opacity: 100%;}
    }
    .loadingStar1 {
        animation: load 0.8s;
        animation-iteration-count: infinite;
    }
    .loadingStar2 {
        animation: load 0.8s;
        animation-iteration-count: infinite;
        animation-delay: 300ms;
    }
    .loadingStar3 {
        animation: load 0.8s;
        animation-iteration-count: infinite;
        animation-delay: 600ms;
    }
